# Next Steps — Make Mnemo Actually Work

> Date: 2026-06-20
> Status: Codebase is clean and restructured. Now build the thing that matters.

---

## The One Thing That Matters

**Memory must persist across sessions without the user doing anything.**

Session 1: Agent makes decisions, fixes bugs, learns patterns.
Session 2: Agent already knows all of that. Zero re-explaining.

This is broken today. Zero memories persist in practice. We fix it NOW.

---

## Architecture (decided)

```
DURING SESSION (passive, never fails):
┌─────────────────────────────────────────────────────┐
│  user-prompt-submit hook                            │
│  1. Read user message from STDIN                    │
│  2. Semantic search memory for relevant context     │
│  3. Output matches as <mnemo-context> to STDOUT     │
│  4. Detect decisions in user message → store        │
└─────────────────────────────────────────────────────┘

AT SESSION END (synthesis):
┌─────────────────────────────────────────────────────┐
│  stop hook                                          │
│  1. Read .kiro/sessions/cli/<id>.jsonl              │
│     (full session transcript — every event)         │
│  2. Extract learnings via pattern matching:         │
│     - Bug fixes (indicator words ≥ 2)              │
│     - Decisions ("decided to", "I'll use", etc.)    │
│     - Patterns ("always", "convention", etc.)       │
│  3. Store extracted memories to memory.json         │
│  4. Create session summary (what was accomplished)  │
└─────────────────────────────────────────────────────┘

NEXT SESSION START (injection):
┌─────────────────────────────────────────────────────┐
│  agent-spawn hook                                   │
│  1. Load hot memories (retention ≥ 0.5)             │
│  2. Load all active decisions                       │
│  3. Load active plan + next task                    │
│  4. Load repo map (compact)                         │
│  5. Output as <mnemo-context> block (≤500 tokens)   │
└─────────────────────────────────────────────────────┘
```

## Execution Plan

### Step 1: Fix user-prompt-submit hook (make search + inject work)

**What**: Every user message → search memory → inject relevant context into agent.

**Current state**: Hook exists but the auto-capture confidence was broken (fixed). The SEARCH + INJECT part may not work correctly.

**Tasks**:
- Read the current prompt-submit.sh generated by hooks/kiro.py
- Verify it: (a) reads STDIN JSON, (b) extracts user message, (c) calls mnemo search, (d) outputs to STDOUT
- Test it manually: echo fake JSON | sh prompt-submit.sh → verify output
- Fix if broken
- Write integration test proving it works

**Success**: When user says "how does auth work?", hook injects any stored memories about auth.

---

### Step 2: Fix stop hook (make session-end capture work)

**What**: At session end, extract learnings from the session and persist them.

**Current state**: Stop hook does pattern matching on the LAST assistant message (from STDIN). This is limited but functional.

**Enhanced approach** (from session doc): Read the Kiro session .jsonl file for the FULL transcript, not just the final message.

**Tasks**:
- Read the current stop.sh generated by hooks/kiro.py
- Verify it: (a) reads STDIN, (b) extracts response, (c) detects learning patterns, (d) stores memories
- Enhance: Also read `~/.kiro/sessions/cli/` most recent .jsonl for richer extraction
- Test: simulate a session end event, verify memories stored
- Write integration test

**Success**: After a session where agent says "the issue was X, fixed by Y", a memory "Bug fix: the issue was X, fixed by Y" appears in memory.json.

---

### Step 3: Fix agent-spawn hook (make recall injection work)

**What**: At session start, inject all relevant context into the agent.

**Current state**: Hook calls `mnemo tool mnemo_recall` and outputs to STDOUT. Should work if mnemo recall works.

**Tasks**:
- Verify spawn.sh: runs mnemo_recall, outputs result to STDOUT
- Verify mnemo_recall: returns compact context with decisions + hot memories + plan + repo map
- Test: run spawn.sh, verify output contains expected context sections
- Write integration test

**Success**: New session starts with agent knowing past decisions and hot memories.

---

### Step 4: End-to-end proof

**What**: Prove the FULL cycle works.

**Test scenario**:
1. Initialize a project (mnemo init --client kiro)
2. Store a memory: "Use PostgreSQL for the database"
3. Store a decision: "Microservices architecture with event bus"
4. Simulate agent-spawn → verify recall contains both
5. Simulate user-prompt-submit with "what database?" → verify memory injected
6. Simulate stop with "Fixed the timeout by increasing retry delay" → verify memory stored
7. Simulate next agent-spawn → verify new memory appears

This is the CRITICAL integration test. If this passes, Mnemo works.
